## 🏗️ Architecture Layers

### 1. Data Collection Layer

**Collectors** gather metrics from both legacy and modern applications:

```
┌─────────────────────────────────────────┐
│           Collectors                     │
├─────────────────────────────────────────┤
│  • LighthouseCollector                  │
│  • APICollector                         │
│  • DatabaseCollector                    │
│  • SonarQubeCollector                   │
│  • CoverageCollector                    │
└─────────────────────────────────────────┘
             │
             ▼
    ┌────────────────┐
    │ metrics.json   │  (per app)
    └────────────────┘
```

**Design Pattern**: Each collector extends `BaseCollector`
- Implements `collect()` method
- Includes validation and error handling
- Enriches data with metadata

### 2. Analysis Layer

**Analyzers** compute comparisons and scores:

```
metrics.json → Analyzers → scores.json
                   │
                   ├─ ParityAnalyzer
                   ├─ PerformanceAnalyzer
                   ├─ QualityAnalyzer
                   └─ MigrationScore
```

**Responsibilities:**
- Compare legacy vs modern
- Calculate dimension scores
- Identify gaps
- Generate recommendations

### 3. Presentation Layer

**Streamlit Dashboard** displays insights:

```
┌─────────────────────────────────────────┐
│         Streamlit Dashboard              │
├─────────────────────────────────────────┤
│  Home → Overview + Overall Score        │
│  Frontend → UI/UX comparison            │
│  Backend → API analysis                 │
│  Database → Schema diff                 │
│  Quality → SonarQube + Coverage         │
│  Performance → NFR metrics              │
│  Summary → Executive report             │
└─────────────────────────────────────────┘
```

**Features:**
- Multi-page navigation
- Interactive charts (Plotly)
- Real-time filtering
- Export capabilities

### 4. Export Layer

**Report Generators** create stakeholder reports:

```
Dashboard → ReportGenerator → PDF/Excel/HTML
                 │
                 ├─ Executive Summary
                 ├─ Technical Details
                 └─ Gap Analysis
```

## 🔄 Data Flow

### Collection Phase

```
1. User runs: ./scripts/collect-all-metrics.sh

2. For each collector:
   ┌─────────────┐
   │ Collector   │
   └──────┬──────┘
          │
          ├─ Connect to app
          ├─ Gather metrics
          ├─ Validate data
          └─ Save to data/{app}/metrics.json

3. Aggregation:
   data/legacy/metrics.json  ─┐
   data/modern/metrics.json  ─┤
                              ├─> Aggregate → data/metrics.json
   browser-agent/parity.json ─┘
```

### Dashboard Phase

```
1. User runs: streamlit run app/main.py

2. Load Configuration:
   config.yaml → AppConfig

3. Load Metrics:
   data/metrics.json → Session State

4. Calculate Scores:
   Metrics → Analyzers → Scores

5. Render Pages:
   Session State → Streamlit Components → Browser
```

### Export Phase

```
1. User clicks "Export" in dashboard

2. Generate Report:
   Session State → ReportGenerator → Template

3. Output:
   reports/migration-assessment.pdf
```

## 📊 Scoring Algorithm

### Overall Score Calculation

```python
def calculate_overall_score(metrics):
    frontend_score = calculate_frontend_score(metrics)     # 25%
    backend_score = calculate_backend_score(metrics)       # 25%
    database_score = calculate_database_score(metrics)     # 10%
    quality_score = calculate_quality_score(metrics)       # 20%
    nfr_score = calculate_nfr_score(metrics)              # 15%
    integration_score = calculate_integration_score(...)  #  5%

    overall = (
        frontend_score * 0.25 +
        backend_score * 0.25 +
        database_score * 0.10 +
        quality_score * 0.20 +
        nfr_score * 0.15 +
        integration_score * 0.05
    )

    return round(overall, 1)
```

### Dimension Scores

#### Frontend Score
```
Feature Parity (60%) + Performance (20%) + Accessibility (20%)
```

Where:
- **Feature Parity**: % of legacy features present in modern
- **Performance**: Lighthouse performance score (0-100)
- **Accessibility**: Lighthouse accessibility score (0-100)

#### Backend Score
```
API Parity (50%) + Performance (30%) + Error Handling (20%)
```

Where:
- **API Parity**: % of legacy endpoints implemented in modern
- **Performance**: Score based on response time vs threshold
- **Error Handling**: Score based on error rate vs threshold

### Gap Indicators

```python
# app/components/comparisons.py
def gap_indicator(legacy_val, modern_val, threshold):
    """
    Shows gap status with color coding
    """
    gap = modern_val - legacy_val
    if gap >= threshold:
        st.success(f"✅ {gap:+.1f}")
    elif gap >= 0:
        st.warning(f"🟡 {gap:+.1f}")
    else:
        st.error(f"🔴 {gap:+.1f}")
```

## 🔌 Integration Points

### Browser Agent Skill

```python
# Integration: Feature parity from browser-agent
from browser_agent import verify_parity

parity_data = verify_parity(
    legacy_url=config.legacy.frontend_url,
    modern_url=config.modern.frontend_url
)

# Store in metrics
metrics['frontend']['feature_parity'] = parity_data['score']
metrics['frontend']['feature_parity_details'] = parity_data['features']
```

## 🔐 Security Considerations

1. **Sensitive Data**: Never commit metrics.json to git
2. **API Tokens**: Use environment variables for SonarQube/GitHub tokens
3. **Database Credentials**: Store in secrets, not config.yaml
4. **Access Control**: Add authentication to dashboard if deployed publicly

## 📚 Development Guidelines

### Adding a New Collector

1. Create `collectors/new_collector.py`:
```python
from collectors.base_collector import BaseCollector

class NewCollector(BaseCollector):
    def collect(self) -> Dict[str, Any]:
        # Implement collection logic
        return {"metric": value}
```

2. Add to `scripts/collect-all-metrics.sh`
3. Update `app/state.py` to use new metrics
4. Create dashboard page if needed

### Adding a New Dashboard Page

1. Create `app/pages/N_📦_NewPage.py`:
```python
import streamlit as st

st.title("New Page")
metrics = st.session_state.modern_metrics.get('new', {})
st.metric("New Metric", metrics.get('value', 0))
```

2. Streamlit auto-discovers pages in `pages/` directory
3. Number prefix controls order (1, 2, 3...)

### Modifying Scoring

1. Edit `config.yaml` to adjust weights
2. Edit `app/state.py` to modify calculation logic
3. Update thresholds as needed
